108  Alternate cryptocurrencies / Announcements (Altcoins) / Re: [ANN][KMD][dPoW] Komodo - Zcash Zero Knowledge Privacy Secured by Bitcoin on: March 22, 2017, 05:02:57 AM
As you know we are offering bounties for finding security issues in the code base. Approx 16 hours ago, we received a vulnerability report.

Within 6 hours we had verified it, developed and tested a fix and started updating the notary nodes. In order to give enough time for the field to update, I set the fix to activate at 4:30 PM UTC. This was about 16 hours time and I was hoping long enough for the field to update and short enough to avoid a zeroday attack.

Unfortunately, the attacker sees the fix on github as it is open source and he deployed an attack within the last hour. I was forced to move up the activation time to 5am UTC, which is less than half an hour from now.

Exchanges have been notified, along with pools. Notaries are almost completed the update. The damage from this attack will be minimal, actually zero, as the security bounty is offset by whatever damage is done, so the total cost remains the same.

If you are running a native komodod, you need to update. The network is about to do a controlled fork.

The problem of fixing things in an open source project is a difficult problem to solve. In this case we deployed a fix within 6 hours of being notified, had everything update and still we get hit with an attack. It does feel like antivirus vs virus.

Anyway, we will get better and better and dealing with these things more efficiently, but it is much better to already have the fix deployed than getting blindsided by an attack. Next time I will set the activation time shorter to give less time for attacker to deploy. (always assume and prepare for next time, so if it happens you are ready)
