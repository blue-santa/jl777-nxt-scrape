OK, that solves the case when there is only one withdraw in a batch
when there are more MGW will actually make a profit from this

here is the batch fee calculation code:
    MGWfee = (rp->numredeems * (lp->txfee + lp->NXTfee_equiv));

lp->txfee is .0001 BTC or .001 LTC
lp->NXTfee_equiv is .0001 BTC or .01 LTC

I just realized there is a very simple fatal attack against MGW!

Just deposit in a tight loop 1 satoshi to MGW multisig deposit address and it will lose 1 NXT per deposit, so it wouldnt take long to drain it of all funds

The only thing I can think to fix this is to charge 1 NXT for deposits. For simplicity, it is set to .0001 BTC and .01 LTC

This makes MGW non-profit (but non-loss) on deposits, but it actually can make a little bit during blocks with multiple withdraws. It is non-profit if there is just a single withdraw to batch

James
