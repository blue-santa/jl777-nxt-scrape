property of sha256 and curve25519

   calc_sha256(0,mysecret,pass,passlen);
    mysecret[0] &= 248, mysecret[31] &= 127, mysecret[31] |= 64;
    curve25519_donna(mypublic,mysecret,basepoint);

when the entropy of the SHA256 output saturates the 256bits, then it wont make it any harder to bruteforce, not that it is easy to do so.

so if you feed in 32 bytes worth of high entropy data, it seems it wont help anymore
openssl rand -base64 64 will generate 64 bytes of base64 (6bits) so this is more than sufficient

43 base64 bytes will map to 256 high entropy bits. of course sha256 will then map these to its 256 bits and then the curve25519 does some complex math to create the pubkey/privkey pair

Now as far as proof goes, without doing anything formal, if the output of SHA256 is high entropy 256 bits then putting in more than 256 bits of high entropy seems to be point of diminishing returns
