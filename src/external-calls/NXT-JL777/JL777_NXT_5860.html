Multigateway++ (jl777) / Re: NXTorrent
« on: April 16, 2014, 05:54:45 am »
http://209.126.71.170/multigateway_NXTorrent_v02.zip has a combined Mac version of multigateway and NXTorrent. For the brave, the source is at https://github.com/jl777/NXTprotocol/tree/master/multigateway

It really needs a lot of testing. It is currently limited to DOGE, I got exactly 1 multisig withdraw to work, so no guarantees you will be able to withdraw. I am not seeing any problems in getting the deposits, so your DOGEs will be safe.

There is one file in the ZIP file, just run it and it will launch a command line program. First thing it does is generate a high entropy password (along with a bunch of files, so best to put it in its own directory). It will print this address at the top of http://127.0.0.1:7777  You need to send it enough testNXT so it can send AM's

You might have to wait a bit and sometimes it returns blank page. just hit refresh. You can see multigateway stats, filter by NXT acct or coin. Before you can make a deposit, you need to generate one and you can set your withdraw address in the same AM.

I have enabled tokenized security,which means all non-display functions will fail at first. It will display the actual command you need with the proper token calculated. copy it and paste it over starting at the ? in the URL.
127.0.0.1:7777/multigateway?<paste the tokenized JSON here>

For the curious, you can look at the commandline window and see a bunch of status going by.

Any crashes, or problems, definitely post here,along with the last part of the command window. I will probably offline most of the day, so dont worry if not response for a bit. It would be nice to get some test results. I am pretty sure the display of account info is not quite right.

James

P.S. NXTorrent does not have tokenized security (yet) and it could use some testing too

« on: April 16, 2014, 01:04:43 am »
Quote from: mczarnek on April 16, 2014, 12:26:28 am
Quote from: Come-from-Beyond on April 15, 2014, 05:54:14 am

I personally would add the ability to pull out more currency by chipping in more Nxt.

What account is supposed to "burn" more NXT?

I say that account that wants to take out more of this currency..  basically you can exchange the two or you can simply put in more Nxt to get out more of this currency.  Like the Nxt cent idea.

I kind of which ChuckOne though, unless you can why is this useful?  I guess maybe for Next Bitcoin or country specific coins similar to Aurocoin (which as predicted, dropped off the charts as soon as the average person was told they had free money sitting in an account somewhere)?
I dont think anybody else has mentioned one fantastic side effect the deflationary version has.
As I understand it X amount of NXT will create Y amount of newcoin. Then no matter what happens, you always have the option of converting the Y newcoin back into X NXT.

Let's say that this newcoin is actually asset backed, eg. multigateway BTC. Assuming that the market price is 20000 NXT for BTC, this newcoin would actually be worth significantly more!!

Let us assume we trust the multigateway to redeem the BTC, so its value should never go below the value of BTC. But, it is also always worth 20000 NXT. This is basically BTC + put option @ 20000 NXT that never expires. It is worth the greater of 20000 NXT or 1 BTC. Some black+scholes values could be used to approximate a multiyear option, but my guess is that it is quite significant.

Now, this value simply appears out of nowhere!!!

That's some pretty powerful feature. It essentially lets us mint options without any risk for any other asset.

Now, I think if the reverse was allowed, eg. burning NXT to get newcoin, that is equivalent to a call option. However, we cant let anybody do it at anytime and there has to be a BTC that is also tied to it.

I am busy debugging multigateway now, so hopefully some of you can figure out how to create pure put and all options with clever combining of monetary system coins with assets.

With clever usage of the NXT -> newcoin -> NXT mechanism,all sorts of powerful things can be created. I hope CfB will implement monetary system to allow for varying the behavior as I cannot figure out all the use cases.

James
