Humbot, if the answer is that the keys on one server are worthless without the other two servers, then I'll go in with you on setting up a triplet if we can find a third operator.   Please understand that it's not personal!
The backups directory (that you have to make) in each coins directory has all the multisig keys from that server.
The way it works is a NXT address creates a bitcoin account with the bitcoind, so now NXT address has its normal bitcoin address on that single MGW server. I currently totally ignore anything that comes into that address and anything sent to this single address is totally under the control of the server operator. just like sending to a deposit address to any exchange.

Now when a generate deposit address request is received via AM on the blockchain (no actual direct contact with users), each MGW server asks the other two the public key associated with the single deposit address. Then all three public keys are used to create a 2 of 3 multisig address. And that is the address people are depositing to.

So, if two of the MGW operators collude, they could empty all the accounts, so one sockpuppet cant do anything, but two of them and danger will robinson!

James
