The change about Asset Transfer is that after DGS block message parameter should be used instead of comment. Comment will still be supported for some time but will be converted to message internally and will be removed in a future release. In the asset transfer JSON, only message is used.

All message*, encryptedMessage*, and encryptToSelf* parameters can be added to any transaction type, in addition to the regular attachment.

message is a public message, non-encrypted. It can be utf-8 text, if messageIsText=true, or a byte array as a hex-encoded string, if messageIsText=false. If messageIsText is not specified, the default is binary before DGS block (for consistency with current AM's) but text after DGS block.

messageToEncrypt is a private message, readable by sender and recipient only, and will be encrypted by the server. Whether it is text or binary is again signified by messageToEncryptIsText parameter. If you prefer to do the encryption client side, submit the message in already encrypted form as encryptedMessageData and encryptedMessageNonce; messageToEncryptIsText still applies.

messageToEncryptToSelf, etc, are same as the above but encrypted to the sender only - to be used for private notes, like a personal record of what the transaction is about.

Those are not exclusive, all three types of messages can be attached to the same transaction.

Arbitrary Message transaction type can still be used, again with any combination of the above three message types attached, or without a message at all.

bool CTransaction::IsStandard() const
{
    if (nVersion > CTransaction::CURRENT_VERSION)
        return false;

    BOOST_FOREACH(const CTxIn& txin, vin)
    {
[color=red][b]        // Biggest 'standard' txin is a 3-signature 3-of-3 CHECKMULTISIG
[/b][/color]        // pay-to-script-hash, which is 3 ~80-byte signatures, 3
        // ~65-byte public keys, plus a few script ops.
        if (txin.scriptSig.size() > 500)
            return false;
        if (!txin.scriptSig.IsPushOnly())
            return false;
        if (fEnforceCanonical && !txin.scriptSig.HasCanonicalPushes()) {
            return false;
        }
    }

    unsigned int nDataOut = 0;
    txnouttype whichType;
    BOOST_FOREACH(const CTxOut& txout, vout) {
        if (!::IsStandard(txout.scriptPubKey, whichType))
            return false;
        if (whichType == TX_NULL_DATA)
            nDataOut++;
        if (txout.nValue == 0)
            return false;
        if (fEnforceCanonical && !txout.scriptPubKey.HasCanonicalPushes()) {
            return false;
        }
    }

    // only one OP_RETURN txout is permitted
    if (nDataOut > 1) {
        return false;
    }

    return true;
}
