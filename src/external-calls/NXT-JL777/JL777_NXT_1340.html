Quote from: jones on December 04, 2014, 05:00:28 am
I was thinking about this and found a way that we can make nxt addresses that dont require initialization without a hard fork or that many changes.

because the account id for an nxtaccount derives directly from the sha256 of the public key, if we use a 62 bit formatting on the public key, it could be used as a valid address.

 There would be two types of addresses, public key form, which is a 43 character 62 bit (uppercase+lowercase+digits) string that would be copy pasted and can directly announce public keys without the user even knowing.

the other type would be the original RS encoded format called "short form" for use with easier readability, when accounts need to be entered by hand, and for use with cold storage accounts.

both would live side by side and be interchangeable, no changes to existing things would need to be made as we are just adding a new format.
since no core changes are needed for this, it can be adopted by websites for internal accounts where the user isnt having to interact with it.
