Online just for a few minutes. What you are saying can already be done!
It is currently needing a recompile, but adding/subtracting a different NXTprotocol handler is as simple as adding one line of code. Assuming you have put everything about that handler into that file.

The following is a main.c that you can use to bypass needing to link with libwebsockets. I test with Xcode, then on Ubuntu command line before trying the libwebsockets version.

char dispstr[65536]; // handlers can write data here and it appears on the webpage instantly via websockets

#include "libwebsocketsglue.h" // includes stub handler
#include "multigateway.h"
#include "nodecoin.h"
#include "NXTmixer.h"
#include "secretproject.h"

int32_t main(int32_t argc, const char * argv[])
{
    init_NXTprotocol();
    register_NXT_handler("multigateway",mp,2,-1,multigateway_handler,GATEWAY_SIG,1,0,whitelist);
    // make sure all handlers are registered
    while ( 1 )
        sleep(1);
}

recompile libwebsockets, relaunch test-server (probably should rename it to something matching what it contains)

Now 127.0.0.1:7777/NXTprotocol will list the installed handlers:

Code: [Select]
{
"numhandlers": 3,
"handlers": [{
"name": "secret project",
"type": 2,
"subtype": -1,
"priority": 1,
"AMsig": "0xdededede"
}, {
"name": "multigateway",
"type": 2,
"subtype": -1,
"priority": 1,
"AMsig": "0xabcd1246",
"whitelist": ["423766016895692955", "12240549928875772593", "8279528579993996036"]
}, {
"name": "stub",
"type": -1,
"subtype": -1,
"priority": 1,
"AMsig": "0x99999999"
}]
}

Not exactly user friendly, but any developer can pick and choose what handlers to include and then make a release with an arbitrary set. Of course, someone with the time should probably come up with a process to enable runtime adding/subtracting of NXTprotocol handlers. For me, it is not a priority, but I guess something like that would be a NXT app store
