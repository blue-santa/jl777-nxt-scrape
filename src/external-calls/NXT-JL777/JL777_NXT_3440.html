The MGW asset was initially not intended to be a giant money maker, at least not in the first year. The lower the fees, the more volume there will be and the better for customers.

The initial fee structure has been set to be free for deposits and a withdraw fee that is enough to cover the tx costs.

Now with uMGW around the corner and possible many people wanting to setup gateways, making money will be part of the equation. I also want to incentivize people to run nodes and while I dont have all the details worked out yet, what I am thinking is that a portion of InstantDEX revenues could be sent to the uMGW nodes. Now InstantDEX is also going with the low fees approach and it will be charging 1/1024'th of the transaction volume (to each side), so we are talking about 0.2% that is available to be shared and half of that is reserved for InstantDEX asset holders. So there is a budget of 0.1% to be split between operating costs (minimal), uMGW nodes, pNXTd nodes, etc.

Anyway, just wanted to post this update as MGW has been thought of more a "non-profit" type of asset, but uMGW really changes all this, especially with the share of InstantDEX commissions

The backups directory (that you have to make) in each coins directory has all the multisig keys from that server.
The way it works is a NXT address creates a bitcoin account with the bitcoind, so now NXT address has its normal bitcoin address on that single MGW server. I currently totally ignore anything that comes into that address and anything sent to this single address is totally under the control of the server operator. just like sending to a deposit address to any exchange.

Now when a generate deposit address request is received via AM on the blockchain (no actual direct contact with users), each MGW server asks the other two the public key associated with the single deposit address. Then all three public keys are used to create a 2 of 3 multisig address. And that is the address people are depositing to.

So, if two of the MGW operators collude, they could empty all the accounts, so one sockpuppet cant do anything, but two of them and danger will robinson!
