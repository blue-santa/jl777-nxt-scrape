Maybe that is better than shred, as shred means assetholders that used to have something, now dont. Ah, so we can have a shred also, just require that it must be the issuer and all assets must be in issuers possesion (or genesis acct). The following would be the replacement process:

1. New asset is created with the same number of assets or integral multiple (we get stock splits from this too!)

2. Magical asset swap is invoked by Issuer so the proportional amount of new asset (with same name) is given to existing assetholders as of a specific block. My dividend calculator already creates JSON for the asset allocation, so everybody can calculate who should get what.

3. Now the issuer has all the assets and can shred.

Alternatively, step 2 can simply send the original asset to genesis, then we dont need step 3.

The "calc dividends" should work, I dont think I broke it with the MGW rewrite, but we should make sure

Both commands accept a maxtimestamp (NXT time) and will display the results as of that time stamp. Its pretty handy if you want to go back to a certain time to calculate dividends or see what transactions were there. I didnt find an easy way to map blocknumber to timestamp. Maybe a side project for someone? Anyway, you can get current timestamp from NXTcore and just subtract N seconds to get the value to put in

OK, now for the MGW commands. The first thing you have to do is request a multisig deposit address using the "Update coin addresses" form. Might as well put in the withdraw addresses (your local wallet) as this is one less thing to do when we test withdraws. You dont need to specify any withdraw address, but then you have to keep specifying an address for every withdraw request. The withdraw address is the backstop address that is used only when the redeem request does not specify an address

Now, this is a VERY vulnerable step as if somebody else can change the withdraw address to theirs, then whenever you redeem DOGE, they would get it! So, just submitting the form is not enough for the "update coin address" command. It will calculate a tokensized JSON pair that proves that the account holder approves this change. A screen should show up with an http://127.0.0.1:7777/... line, just copy the whole thing and paste it into the browser URL. If it works, it should display the txid for the AM it just posted.

[make sure you have funded your acct! otherwise you cant do any of this. When longzai gets the GUI working with the forms, I can make it so that the acct used is the one the user types in]

When the next block comes in, you should see the AM printed in the commandline window. The MGW servers will also see this and they will have a conference call (not really, but similar) and jointly create a multisig deposit address for you. This address is bound to your NXT acct and when you send DOGE to it, it will automatically send DOGE assets to your NXT acct. Also, it will stay in that acct and not get moved to a central sweep acct. This is the big vulnerability with central exchanges as the big central acct gets to be very tempting. With MGW, the acct starts with what you put into it, but over time all accounts will end up having the same average amount. So, other than being a place for you to deposit, it also forms part of a virtual MGW unspent outputs. This means funds could come out of your deposit address to satisfy someone else's withdraw request. This is OK, as there is always 1:1 correlation of deposits and unredeemed assets

So, for now, that is really all there is to test. Make address, send DOGE, wait 15 minutes and make sure it arrives in your NXT acct.

The "Display account details"  and "Display coin details" are somewhat legacy, but should spit out a lot of info.

Oh, to have your computer track the MGW deposits, you do need to have the DOGE wallet installed and properly configured.

server=1
txindex=1
