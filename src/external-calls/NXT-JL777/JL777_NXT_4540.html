both parties need to submit an AM with identical parameters (mirrored) and when both sides match up to each other they do a swap of txA with txB

txA and txB can be anything

They need to agree that txB is the 100000 NXT payment to Alice and that txA is the invoice AM from alice to bob.
both submit the NXTatomic AM that specifies all this and when both nodes agree that everthing is in order
the invoice AM is sent and triggers the payment AM

It is possible to get IP addresses and link them to NXT accounts, but only if a node chooses to do so

Inherently NXT is far more private than BTC due to not having txin/txout links that go all the way back to genesis block. With NXT, once funds go into an account, they are all fungible. « on: June 12, 2014, 07:08:11 pm »

I don't know too much about internet privacy... especially when it comes to cryptocoins, but I do know a secure server/java library for generating true random numbers through radioactive decay. It is called Hotbits and I am sure it could be implemented some way to help secure privacy.

www.fourmilab.ch/cgi-bin/uncgi/Hotbits?nbytes=128&fmt=bin

If this RNG became the default implementation and was from a central authority then you are adding more security issues than you will be improving.

What if we decentralized it? A number of computers using the NXT client each generate a series of random numbers through hot bits. These random numbers are then shuffled in an array then a psudo-random number generator picks one number from the array and uses it. That way the numbers are coming from all over the place but are still taking advantage of the true-random numbers.

Another (pretty good) method of generating random numbers is to use white noise. Perhaps we use a sample of white noise from every computer running the NXT client (like one millisecond so as not to intrude on privacy) and generate a random number from that. It would be impossible to predict which number would be chosen unless someone can somehow monitor the sound from every single computer running the NXT Client. That seems like a more secure way to generate a random number if you don't like hotbits.

Ok, so who is willing to implement this? and why again? entropy is imortant, i get it, but discussing all these overkill solutions still doesnt brings us any further in the Problems we want to solve - NXT Privacy.
having all NXT nodes participating is a BIG part of NXT Privacy
Think about it in real world terms.

you are being followed by some bad guys and you want to lose them
a) you go to an place that has very few people
b) you go to Times Square at midnight New Years eve just as all the balloons are falling

which makes you feel safer?
