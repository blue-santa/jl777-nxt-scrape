Nxt General Discussion / Re: NXT blockchain as a decentralized data storage system?
« on: October 22, 2014, 04:35:26 am »
Quote from: jones on October 22, 2014, 04:02:20 am
I'm also doing a write up for decentralized storage using the nxt blockchain.

I have somewhere around 5 pages in a notebook so far. I really liked the whitepaper of Sia, its too bad that it has run into pitfalls with the quorum setup.
my idea is to store the users files as a small database, from there each user file system is individually dealt with and stored in the network across many nodes.
from there you just need a system of downloading and check summing the files and your one step closer to decentalizing the internet. 
Would you like to help me?
I already have most of what you are talking about already coded. basically each node stores the DHT packets it is closest to and all this is in a BDB database.

./b SuperNET '{"requestType":"getdb","contact":"handle","id":0,"dir":"send"}'
./b SuperNET '{"requestType":"getdb","key":"DHTkey","dir":"send"}'

you can access the fully encrypted user <-> user comms from the private.db or the public.db which just has DHT keys

I also have M of N save and restore calls:

./b SuperNET '{"requestType":"savefile","filename":"<filename>","L":0,"M":2,"N":3,"password":"<pass>","pin","0123"}'
./b SuperNET '{"requestType":"restorefile","filename":"<filename>","L":0,"M":2,"N":3,"password":"<pass>","pin","0123","destfile":"dest","sharenrs":"<fromsave>","txids":[<fromsave>]}'

So the checksumming is automatic using M of N as any bit error and it wont reconstruct. You avoid sybil attacks or at least make it really hard as only the one that saved the file will get the random "sharenrs" values along with the txid array. both are needed to properly reconstruct. As far as anybody else is concerned, there are just random fragments in the cloud.

These M of N fragments automatically go into the public.db
It is still quite new code and could sure use some intensive testing from someone that understands what it is doing at the detailed level.

James

P.S. By not password protecting a file, you basically are publishing, of course you need to also publish the sharenrs and txids, but that can be in another file, so eventually a list of txids would all be encapsulated in a single short identifier. If that can be mapped to alias, then using alias decentralized files can be accessed
