Alias System / Re: Alias transfer
« on: May 13, 2014, 11:44:26 pm »
Quote from: kushti on May 13, 2014, 11:44:42 am
Initial testing shows good results so I have sent code to JLP for review and going to write more tests.

Interface details:

Overall
There are two actions over blockchain transactions: SellAlias & Buy Alias. SellAlias kinda transaction has 3 fields supporting sell operation: alias, price and buyer id.
Set buyerId to Genesis.CREATOR_ID if you would sell alias to everyone, or you can sell only to chosen account by providing it's id as buyerId. BuyAlias tx has only 2 fields: alias & price. Sell & buy orders match only if price in BuyAlias operation >= price in SellAlias.

Java API
We have 2 more attachment (sub)types MESSAGING_ALIAS_SELL & MESSAGING_ALIAS_BUY. ALIAS_SELL has 2 fields: alias and price, ALIAS_BUY alias field only(tx's amount field being for price).

Alias selling for 12 NXTs example(Scala code, but understandable for an every Java guy I guess):

Code: [Select]
      val sellerPhrase = "super secret phrase"
      val priceNQT = Constants.ONE_NXT * 12
      val attachment: Attachment = new Attachment.MessagingAliasSell(alias, priceNQT)
      val publicKey = Crypto.getPublicKey(sellerPhrase)
      val recipient = Genesis.CREATOR_ID
      val tx = Nxt.getTransactionProcessor.newTransaction(deadline.toShort, publicKey, recipient, 0, Constants.ONE_NXT, null, attachment)
      tx.sign(sellerPhrase)
      Nxt.getTransactionProcessor.broadcast(tx)

Alias buying for 15 NXTs example:

Code: [Select]
      val buyerPhrase = "another super secret phrase"
      val publicKey = Crypto.getPublicKey(buyerPhrase)
      val recipient = Convert.parseUnsignedLong("3571922012891302654")
      val attachment: Attachment = new Attachment.MessagingAliasBuy(alias)
      val tx = Nxt.getTransactionProcessor.newTransaction(deadline.toShort, publicKey, recipient, Constants.ONE_NXT * 15, Constants.ONE_NXT, null, attachment)
      tx.sign(buyerPhrase)
      Nxt.getTransactionProcessor.broadcast(tx)
Http API
There are two methods: sellAlias & buyAlias. Both are sharing common tx fields ("secretPhrase", "publicKey", "feeNQT", "deadline", "referencedTransactionFullHash", "broadcast"), sellAlias has additional fields ("alias", "recipient", "priceNQT"), buyAlias has additional fields ("alias", "price").

Application layer
So what you can do on top of this?
  • Simple alias transfer - just issue sell order with price=0 and appropriate recipientId and then buy order from that Id with price=0.
  • Public alias sell - just issue sell order tx with desired price and recepientId=Genesis.CREATOR_ID(I would like to call it GOD_ID) and wait for buyer
  • "Aliases for sell" list services - just scan blockchain for non-matched and non-expired sell orders
  • Alias auctions - you can use referenced txs or other mechanisms for escrow etc
