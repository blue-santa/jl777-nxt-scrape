it is the "Authorization Committee" as in the request has been approved but needs to be authorized.

The basis for accepting/rejecting a funding request is by verifying that there is a valid spending request made under the then current rules for SuperNET. This means that even if you disagree with the business decision, if the request conforms to the rules, then it should be approved.

This separates the decision making from the ability to approve from the ability to actually send the funds. A threeway split of power.

For spendings decided by voting, it should be quite simple, eg. just make sure the vote result tally is valid and since such votings will be on the blockchain, I imagine there will be some file with the details of the tally.

For spendings like the funds used for purchase/swaps of new coins, the judgement needs to be made as to whether the deal falls into that category and not whether you are agreeing with the exact price negotiated, etc.

The purpose of the Authorization Committee is to make sure that there is no rogue spending attempt for things not approved by SuperNET. I think this should make it much less of a time requirement as we are needing to eliminate fraudulent spending requests and not applying subjective judgements on the merits of each spending request.

The assumption is that if a spending request has made it through the SuperNET biz processes to reach a formal request for funding that it is making business sense.

James
