3598  Alternate cryptocurrencies / Announcements (Altcoins) / Re: [ANN] BitcoinDark (BTCD)--Sha-256/PoW-PoS hybrid/Bounty Opportunities on: August 18, 2014, 01:28:16 AM
http://209.126.70.170/teleport_test0

The above is a DEBUG version of BitcoinDarkd for Ubuntu. It runs on mainnet, so be very careful. I need just one or two other nodes. You need to make a jl777.conf file in the same directory as teleport_test0

Code:
{"MAINNET":1,"MIN_NXTCONFIRMS":13,"active":["BTCD"],"coins":[{"name":"BTCD","maxevolveiters":10,"useaddmultisig":1,"nohexout":1,"conf":"/home/<username>/.BitcoinDark/BitcoinDark.conf","backupdir":"/home/<username>/backups","asset":"11060861818140490423","minconfirms":1,"estblocktime":60,"rpc":"127.0.0.1:14632","clonesmear":1,"srvpubaddr":"<BTCDaddr>","pubaddr":"<pubBTCDaddr>","privacyServer":"127.0.0.1"}]}

you need to make two addresses, one for your public BTCD address "pubaddr" and one for your privacyServer's public address "srvpubaddr". any address that your BitcoinDarkd knows about is fine. Also change the <username> to your username, it will use the path to check for the BitcoinDark.conf file and also to make backups. You might need to manually create the backups dir.

This is only to test that the new BitcoinDarkd runs on a system that is mine and initalizes properly. If all goes well, I should receive a pubkey broadcast and then be able to attempt teleports to your node. Most likely will need many more versions to allow proper reception of telepods, but gotta start somewhere.

Since RPC is not hooked up yet, you cant do much other than make sure it doesnt crash and initializes properly

James

P.S. It is quite verbose so I can see exactly what is going on. Also, you need NXT running on mainnet also, but no need to fund any accts, just need it running
