Thanks to BTCDdev, the error -22 was solved in a few minutes! its nice to have someone who is able to deal with C++ code, I really am not so good with C++. I think the integration of libj777 will go quite smoothly without some barbarian like me blundering through the C++ code. We have the RPC enhancements defined and this will allow not only the QT dev to invoke any of my core API, but also any user could from the debug console.

Anyway, the change will require some fundamental changes to the coin protocol and i think this will be a pretty significant barrier for any existing coin to try to get Teleport directly. We basically have to make Teleport enabling changes to the inner core of BTCD. Any other coin that does that basically admits Teleport is the right solution and then the obvious conclusion for users is to just use the original Teleport coin. This combined with the clone disruptor/disintegrator should give us quite some time before we get copy cats. Another big disincentive for the clones is that they give up any revenue sharing from the biggest realtime decentralized exchange and who would use a clone's network when it has so few bid/asks on anything? In addition to the tech, the network and userbase will be a competitive barrier.

If only we had a bunch of other new stuff coming out in the next few months, oh wait! We do

No need for buying panic, yesterday had a bit too high a bump, the 5% per day is the sustainable base, anything faster could destabilize our ascent to the top tier of coinmarketcap
