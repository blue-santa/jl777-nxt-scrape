Notice to exchanges only:
(If you are not running komodod for an exchange, ignore this post)

The following is a suggested way for exchanges to switch from not having -exchange set to having -exchange set. If you never ran komodod without -exchange, then there is no need for this migration method

To change modes:
a) backup all privkeys (launch komodod with -exportdir=<path> and dumpwallet)
b) start a totally new sync including wallet.dat, launch with same exportdir
c) stop it before it gets too far and import all the privkeys from a) using komodo-cli importwallet filename
d) resume sync till it gets to chaintip

For example:
./komodod -exportdir=/tmp &
./komodo-cli dumpwallet example
./komodo-cli stop
mv ~/.komodo ~/.komodo.old && mkdir ~/.komodo && cp ~/.komodo.old/komodo.conf ~/.komodo.old/peers.dat ~/.komodo
./komodod -exchange -exportdir=/tmp &
./komodo-cli importwallet /tmp/example

The above process will migrate all privkeys from the old regime of no -exchange to the new one with -exchange.

coinexchange.io is helping to validate this process works as expected. Thanks to him for all the help in getting this fix and migration process validated
