certainly, every network has some sort of upper limit.
for now we are running at less than 1% capacity, actually closer to 0.1%, but we are already preparing how to deal with saturation of the first barterDEX p2p network.

The answer is to spawn new ones that are isolated from each other. then there is no practical limit, the only issue is that getting the absolute best price available globally becomes a bit more difficult, but since we are talking about saturation level situations, that means there is no shortage of liquidity makers.

by limiting each p2p network to certain trading pairs, that would mitigate the global price issue.

Now that the basics are working, the next issues are about scaling. For all that are saying it needs to be ready for grandma, that assumes that scaling to a million users is already solved.

It isnt, yet. I dont see any reason new p2p networks cant be created to match the volumes. There will be some amount of turbulence, but unlike when a central exchange goes down, with a p2p exchange it would just make some nodes not available, or some of the p2p networks will be bogged down.

Having all of them go down all at the same time would mean the internet went down. Of course getting the reality to where the theory says it should is what 2018 will be about.
