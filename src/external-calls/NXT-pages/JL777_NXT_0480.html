It is my impression that lyaffe's negative comment referred to a larger change also suggested in this thread, which is not what me and capodieci are talking about.

It is complex, the problem is that currently the code validates the transaction fee immediately when a transaction is submitted, before it even enters the unconfirmed pool. At this time we don't have enough information to tell if a currency can be exchanged, for example, since the exchange offer is still unconfirmed or the user can still change the offer before the block is applied. Then when the transaction is applied we'll have to reject these transactions in case the fee is not covered but we'll have to keep them unconfirmed since they may be able to pay the fee in the future if an exchange offer is published.
Two more complications are phasing and adding an, if (currency sell) do this, statements in several key places to make sure the fee is checked after the balance has been updated.
