SuperNET Releases / Re: uMGW - release of Universal MGW daemon
« on: July 22, 2014, 05:16:35 am »
Quote from: longzai1988 on July 22, 2014, 05:03:31 am
Quote from: jl777 on July 22, 2014, 12:57:14 am
Now that we are close to having running uMGW servers, we need a way to request deposit address and withdraws:
I am hoping that longzai could make a uMGW page where the gateway_sig, issuer acct and coin are configurable, otherwise someone who is not me will have to generate AM's that are mixed binary + JSON, fun!

James

1) request deposit addr - by sending an AM, format as follow

struct NXT_AMhdr
{
    uint32_t sig;
    int32_t size;
    uint64_t nxt64bits;
};

struct json_AM
{
    struct NXT_AMhdr H;
   uint32_t funcid,gatewayid,timestamp,jsonflag;
    union { unsigned char binarydata[sizeof(struct compressed_json)]; char jsonstr[sizeof(struct compressed_json)]; struct compressed_json jsn; };
};

the sig is: #define GATEWAY_SIG <your gateway sig>
size is the size of the total AM so it changes based on len of JSON
nxt64bits is binary of sending NXT acct
funcid should be 'g'
gatewayid doesnt matter, best to set to 0
timestamp is the NXT timestamp
jsonflag should be 1

example : https://nxtblocks.info/#section/txid/11975465686591368054

2) asset withdraw

using NRS core transfer asset api with additional comment following the below json:

{"redeem":"BTC","withdrawaddr":"<BTC_ADDR>","InstantDEX":""}

example : https://nxtblocks.info/#section/txid/3870687962511378450

Meanwhile we are testing against one dogecoin correct? If that is the case I can duplicate very quick to test request deposit address and withdrawal

p/s : In near future mgw lite core code need to re-factor to allow better scalability
I think for easy and quick, why not just make the key variables easy for even non-HTML types to change.
Not sure if you can make a .conf file like I did for MGW.conf, but if there could be a MGWgui.conf file to set the issuer, coin name, key accts, etc. then you wont have to make a zillion versions, each uMGW triplet can just edit the MGWgui.conf file

If a .conf file is not possible, then maybe just having some variables in an easy to edit place so people who dont understand HTML, but know what things need to be changed can do it
