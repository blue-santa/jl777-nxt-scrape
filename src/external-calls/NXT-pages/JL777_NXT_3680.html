hey all. so i've been dabbling for the past couple of days and realized just now the best security for securing your passphrase for nxt is the following:

Since we can edit the html/css files of nxt, we can technically cloak the entire frontpage interface to our own setup. This would in effect defeat keyloggers and trojans.

- edit your hosts file and reroute a site you don't really visit to the nxt client page (127.0.0.1:7678). keyloggers would think your visiting blahblah.com instead of 127.0.0.1

- most of us save our passphrase onto file and we copy and paste it out, but trojans can record your clipboard data. A solution to this is, embed the passphrase into a javascript file that pops up on the page where you DRAG and DROP the passphrase into the text field. This bypasses copying anything into the clipboard.

- Since the javascript file exposes your passphrase, add in obscure words/characters to your passphrase to eliminate anyone just copying it. they would also need to know what doesnt belong there.

so the only way a hacker can steal your passphrase is if they had a screen recorder trojan. then nothing can really stop that.
I think keeping vast majority of funds in offline computer that does offline signing is the best way.
the offline computer only does this and NEVER installs anything else. Never goes online. Never installs anything. Then should be safer than any solution where you are messing with passwords on a machine connected to live Internet
