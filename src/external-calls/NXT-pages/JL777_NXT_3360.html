https://nxtforum.org/job-board/(javascript)-pay-with-nxtbutton/msg63688/#msg63688

One way to do it would be to have a scheme that centers around actions rather than accounts.
This could be as simple as using existing API function names, like this:

nxt://requestType?param=xyz&param2=123

Where "requestType" is an API function name.

Some examples:

nxt://sendMoney?recipient=NXT-9SDF-57KE-H8RU-9BHWY&amount=123
nxt://generateToken?website=http%3a%2f%2fgoogle.com
nxt://setAlias?aliasName=MyName&aliasURI=MyData
nxt://sendMessage?recipient=NXT-9SDF-57KE-H8RU-9BHWY

Think asset exchange and marketplace and voting etc. functions too.

For instance the generateToken function does not make much sense in a account centered context because there is no account number to enter.

Everyone talking about widespread adoption and Nxt being a platform for all things... this (relatively small) feature could be a step in this direction. We just need to agree on a common standard.

I'm all for adhering to existing standards too, but Nxt is new so there are no existing standards for what we need.

Any thoughts?
