CFB doesn't want to do it (or doesn't have time)

We just need a list of all possible phasing scenarios.
I just need a mechanism to allow blockchain enforcement of all or none of the linked tx that is immune to mczarnels' evil forger attack.

One approach that I think would be relatively simply is to allow a singled nested tx:

Alice makes ANY type of tx and gives the unsigned txbytes + sighash to Bob (fullhash is calculable from this), now Bob makes ANY type of tx and includes Alice's txbytes + sighash as part of the attachment, "linkedtxbytes" and "linkedsighash" fields. Now Bob sends back to Alice this nested TX with his unsigned txbytes and sighash for Alice to verify that it is satisfactory.

As creator of the nested tx, Alice now has the ability to trigger both parts of this tx as long as it is done within timeout, so to avoid being held hostage for long time, it is suggested both sides put in timeouts of ~1 hour. Not sure the mechanism for Alice to trigger the nested tx, maybe just the submission of the signedtx bytes that matches the sighash?

Now if there was a way to cascade such things, then an arbitrary number of tx could be made to all execute, or none at all. Of course at some point it wouldnt fit into a block, but maybe there would be a "connector" txtype that simply got a bunch of fullhash references and it would trigger either all of them or none of them get executed
