    static char *sendfrag[] = { (char *)sendfrag_func, "sendfrag", "V", "pubkey", "name", "fragi", "numfrags", "ipaddr", "totalcrc", "datacrc", "data", 0 };
    static char *gotfrag[] = { (char *)gotfrag_func, "gotfrag", "V", "pubkey", "name", "fragi", "numfrags", "src", "totalcrc", "datacrc", 0 };

The sendfrag and gotfrag are working and are the main send and ack back and startxfer spawns a new task that loads a file into memory and loops on sendfrag, waiting for the gotfrag, resending if nothing back some reasonable timeframe, etc.

I havent decided how to store this yet as the prime use case is for MGW signed withdraw tx, which is always in RAM, but come to think of it, I do make a backup of it, so maybe I will just make these into files on the destination side.

Bitcoin is the only crypto currency being used as money, the large majority are used as crowdfunding/shares for applications or scams. Even LTC or Blackcoin are a tiny portion of the already relatively small amount of merchants that accept BTC and with network effect, not sure that is going to change.

This is specially true since the large majority of mobile and user facing applications are for BTC, NXT has one mobile app right now, even if we had more, the market is still saturated. Perhaps a SuperNET mobile app will change this though as it offers many times more value than single coin mobile apps. In which case, NXT would be in the background as other coin technologies are used.

PS: I probably don't understand what MS can bring to the table.
InstantDEX'ing of asset <-> asset will be at least 2 NXT trades, maybe 3, so that is txfee x 3
You know how SuperNET made the clone coins marginal? MS will make them totally dead
