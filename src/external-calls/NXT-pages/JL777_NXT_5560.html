As I am a not technican. I was wondering why all these new services will need NXT. What makes it unique.  How can an investor in NXT be sure that these services will not be moved to another blockchain, after a price rise of NXT?
See also Buffet discussion e.g. intrinsic value.
1 minute blocktime and largest Proof of Stake network give NXT fundamental advantages. Now that assets are going to be issued, those also create a barrier against clones and other blockchains.

While it is true that technically, any NXT based service could go to a clone or simply fork NXT and run a private network, but then it creates a perpetual catchup and merging workload. As NXT continues to gain core features, new services and assets, etc., it will become impractical if not impossible to keep up with NXT. Most services will not be open source, so it would be like being able to copy a car's engine, but the rest of the car needs to be built. Thats a lot of work

Network effect will kick in and all the top tier businesses will be on the NXT blockchain, and new businesses will go to the largest network. Positive feedback loop

SuperNET Releases / NXTservices v03, includes multigateway deposit support for BTC, LTC, DOGE
« on: May 11, 2014, 12:20:47 pm »
http://209.126.71.170/NXTservices_v03.zip
For people with win8, you might need the files in http://209.126.71.170/win8extras.zip

There is no need to disable enforcement of POST in the .conf file anymore. It should work with the default NXT core install.

This is still early release, so dont use a lot of real money. The multigateway form is working for deposit and status. Withdraws are disabled until I can test it a bit more. When you open 127.0.0.1:7777 you need to do the initial account funding, etc. from the v00 release. Then you will be able to get a multisig deposit address generated by putting in the withdrawal address for the respective coins

You will have to wait for a couple of blocks for the deposit address to come back via AM. It should appear in the top of the form. I have noticed some glitches with the html on windows, but it could just be due to my running win7 under parallels. I think that is why I couldnt UDP punch from my mac to win7 as they are on the same hardware.

OK, so when you submit the multigateway form, it will return a page with the same data you put in, but with a security token all put into a nice URL that you can cut and paste into the browser URL. This is so that other people cant do money actions. For example, if we didnt have the tokenized step, anybody can change the withdraw address and when you do a withdraw, it will go to someone elses acct. I want to prevent that, so it takes an extra cut and paste step. When there is a real GUI on top of this, then it would be automatic.

After you get your deposit address, whenever you send funds to that address, your NXT acct will get the corresponding NXT AE asset.

The NXYsync chat, NXTorrent and NXTsubatomic are mostly unchanged from v00

James

P.S. I hope to have a withdraw enabled version done soon. Funds you deposit will be fine, but only deposit what you are willing to lose (or donate)
