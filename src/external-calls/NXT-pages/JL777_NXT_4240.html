Centralized exchanges have both a gateway + exchange. If you go to the finance/account page you will see the deposit/withdraw form where they give you a deposit address and you give it a withdraw address. So MGW is essentially the finance/account page equivalent.

In a centralized exchange, the deposits turn into internal database entries at the central exchange. They then have some sort of GUI that lets people make bids/asks trades which changes the balances in their database, along with the commissions for each trade. Does the total amount in their database match what is in their actual wallets? Where is there wallet? Can we see it? Fractional reserve? Who knows?

The NXT AE replaces the centralized exchanges GUI/database for trading. And unlike the centralized exchange where we need to trust their database's proper operations, with NXT AE we all see all the trades and with sites like nxtreporting.com it gives even better visibility.

So, my conclusion is that (NXT AE + MGW) > centralized exchanges as far as transparency and reliability goes.
The only issue is that NXT AE takes a minute and sometimes 10 minutes to update to next block. If only there was someway to get realtime trading added to (NXT AE + MGW)...

Last question first. If you dont specify a withdraw address for BTC where should it go? Hey, maybe we could make that a nice way for people to donate to MGW

It is not exactly intuitive on how the accts work. If everybody has their own custom multisig acct, it will be natural for people to think that all the funds there are theirs. However, this creates a problem if somebody has 100BTC in their acct. One of the goals of MGW is to prevent there from being any large accts. hackers drool over 100BTC, but 1BTC is probably (hopefully) not worth their time.

So I had to decouple the amount in "your" deposit address from your balance. NXT AE is keeping how much BTC you have, so ALL of the deposit addresses create a single "virtual wallet". There is no need to sweep things into a central wallet, I use some magic tricks so that I can use the same set of addresses as deposit addresses and part of the global virtual wallet at the same time

At the global level, the sum of all available funds (credits) needs to match the sum of everyone's assets (debits), this is calculated and printed if you have connected your bitcoin/litecoin wallets to NXTservices with these lines:

get_confirmed_deposits BTC returns 61 0.16157016 vs assets 0.21328540 diff -0.05171524
get_confirmed_deposits LTC returns 19 0.77439676 vs assets 0.96810000 diff -0.19370324

If you dont have the bitcoind/litecoind configured for NXTservices, it wont be able to calculate the first parts:
BTC returns 61 0.16157016 and LTC returns 19 0.77439676

hope that makes sense.

tl;dr each multsig deposit address simlutaneously acts as a private inbox for incoming deposits and part of a global virtual wallet for all of MGW and somehow MGW magically makes sure everyone's balances is exact so the global total balances are exact
