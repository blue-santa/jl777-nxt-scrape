The line count is creeping up...
I now have 12 files, just for the multigateway handling. This doesnt count the other 10 for NXTprotocol, nodecoin, etc.

Now that I reached code complete, it was time to totally rewrite it again

The need for remote nodes to run the multigateway wasnt exactly planned for and that broke an internal assumption, eg. that each computer running multigateway had access to the coin daemons. Now, there are three cases:

A) Full multigateway
B) Validating nodes, does almost all the Full multigateway does, expect for creating new accounts and sending money
C) Normal nodes, which just track things via AM and other NXT blockchain events

In order not to go insane, I need to make sure the same code is run in A) as in C) for the same function. The worst thing is to have two (or three) versions that do pretty much the same thing, but its in different places. One "harmless" change in one place and not the other and you end up debugging for hours.

So, this trigged a major rewrite. The good thing is that the low level code is porting intact, just the messy higher level code that ties it all together. Once all the redundancies have been squeezed out of the code, it becomes much harder for any bugs to remain hidden

James

The API for NXTprotocol and handlers is through websockets.

http://127.0.0.1:7777/ will actually display a webpage with buttons and the most recent API call and result. This page will be replaced by Wesley's GUI

http://127.0.0.1:7777/NXTprotocol will list info about all the installed protocols

http://127.0.0.1:7777/multigateway will dump all the info about the multigateway state, including all NXT accts (including all assets and asset trades) and all coin deposits and withdraws.

http://127.0.0.1:7777/multigateway?{"requestType":"dispNXTacct","NXT":"11445347041779652448"}
will display all info about the NXT account, includes all coin accounts, assets and asset trades

http://127.0.0.1:7777/multigateway?{"requestType":"dispNXTacct","NXT":"11445347041779652448","coinname":"DOGE"}
will display info about the NXT account, includes all DOGE accounts, assets and asset trades

http://127.0.0.1:7777/multigateway?{"requestType":"dispNXTacct","NXT":"11445347041779652448","assetid":"13606501668992980954"}
will display info about the NXT account, includes all coin accounts and trades of asset 13606501668992980954

http://127.0.0.1:7777/multigateway?{"requestType":"dispNXTacct","NXT":"11445347041779652448","coinname":"DOGE","assetid":"13606501668992980954"}
will display info about the NXT account, includes all DOGE accounts and trades of asset 13606501668992980954

http://127.0.0.1:7777/multigateway?[{"requestType":"genDepositaddrs","NXT":"11445347041779652448","coins":{"DOGE":"DNyEii2mnvVL1BM1kTZkmKDcf8SFyqAX4Z"}},{"token":"bd855r204lqfskfg4que97uef8sh2blnh7ol9mcq2mmee1i2lu1c2ftnblt08e805kvcu3o6ror0lfuvevuagdfjnlnfieg4jlfshtp1ca9ge9cvuvdh4tlehjqtln27co314dc696ise041msviislavr1i6cgv"}]
You have to specify a withdraw address, even if it is just ""
You can list more than one coin. The token needs to be generated by specifying the first part {"requestType":"genDepositaddrs","NXT":"11445347041779652448","coins":{"DOGE":"DNyEii2mnvVL1BM1kTZkmKDcf8SFyqAX4Z"}} as the website token

Once you issue a genDepositaddrs, you can use the dispNXTacct to get the deposit address
Once you have a deposit address, any transfers into that address will automatically credit your NXT Asset balance
You can then transfer the asset back to multigateway and it will default to crediting your withdraw address
