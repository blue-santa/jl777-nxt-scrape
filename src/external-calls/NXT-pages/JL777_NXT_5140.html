It's becoming clear that this seemingly minor issue actually connects to some larger issues. The discussion seems to have grown from one on how to display assets to how to validate assets and ensure they are legit. This second question has aspects that go way beyond the display issue. This seems like a difficult problem with any distributed exchange, and it seems that ultimately it's a problem of verifying accounts (i.e. associating them with real-world users) and the difficulties that creates for the task of measuring the actual trade levels of an asset. It seems that some kind of reputation system is required; but a rep system seems much more difficult to have for Nxt than for, say eBay. The issue is that any individual can easily create an unlimited number of accounts, thereby creating an inflated image of an asset's trade volume, trade frequency, positive reputation feedbacks, or just about anything.

The only solution to this problem I think can work is the use of stake to weigh the feedback coming from an account (again, the fairness issue. Well, life isn't fair). But even this could be gamed, since one can create a new account, move one's Nxt to the new account, and give multiple feedbacks backed by high stakes in this way. Don't know, but this seems like an important issue with no easy/simple fix.

not if you score the stake including the account creation time + maturity of the stake. there are serveral key values you can check.
the data is there, the hard part is to find the best working weighting combination. but as said, there is no 100%  but you can come
close to it. personally i wouldn't trust any 'human' created list. latest with volume this leads into trouble. pure math and the blockchain
could solve this. best would be a transparant scoring algo, easy to understand and to reproduce, adjusted to it's needs.

Yep it seems that would be key data in verifying stake for these purposes. I think a solution like this would allow for a higher degree of decentralization than any man-made lists.
If you did a path (graph) analysis of the accounts of all the assetholders, you can get a pretty good idea of how many independent asset owners there are. Basically the distance of the NXT that was used to purchase the asset from all the other purchases creates an N-dimensional vector. Ideally you would calculate an N-dimensional distance but the approximation calculated sqrt(sum(Di * Di)) will probably be a decent enough metric. Di is the NXT distance

So the tricky part is calculating the NXT distance between two payments. Too tired now to think this through but it feels like it should definitely be possible to automatically calculate relevance metric based on the number of independent asset holders.

Increase the relevance further by weighing the accts by how much NXT they have and it becomes super difficult to game.
